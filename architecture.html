The Problem

Distributed teams across Nairobi, Lagos, and New York handle customer conversations through calls, emails, and chats — but no one has a unified view. Compliance gaps go unnoticed, patterns repeat across branches, and agent performance is invisible.

The Solution

TraceConnect reads vCon conversation records from a shared database and presents a Conversation Radar — pattern detection across branches, consent tracking per local law (Kenya DPA, Nigeria NDPR, US TCPA), performance rankings, and an in-app smart search sidebar.

Under the Hood
How It Works

🔌 Shared Database, Zero Duplication

The vCon MCP Server and TraceConnect point to the same Supabase PostgreSQL instance. Rails maps its models directly to vCon tables — no ETL, no sync jobs, no data copies.

config/database.yml url: ENV["DATABASE_URL"] # Same Supabase connection string # as the vCon MCP Server

🧠 Pattern Detection

The Radar automatically scans issue categories across all branches. When the same issue appears in 2+ branches, it fires a Pattern Alert — e.g., "product_defect reported in both Lagos AND Nairobi."

category_branches.select do |cat, branches| branches.keys.size >= 2 end # Cross-branch pattern = systemic issue

📜 Consent as First-Class Data

Each vCon carries consent metadata: consent_given, consent_law, consent_expiry_date. TraceConnect checks expiry against local privacy laws (Kenya DPA, Nigeria NDPR, US TCPA) and blocks expired records.

{ "consent_given": true, "consent_law": "Nigeria NDPR", "consent_expiry_date": "2026-03-01" } # Expired -> grayed out, AI blocked

🔍 Ask Sidebar — Smart Search

The sidebar uses a QueryParser that extracts intent + filters from keywords. No external AI API — pure Ruby pattern matching with synonym maps. Results stream back via Turbo Streams.

"compare Nairobi and Lagos billing" -> intent: :compare -> branches: [Nairobi, Lagos] -> category: billing -> Turbo Stream append response
